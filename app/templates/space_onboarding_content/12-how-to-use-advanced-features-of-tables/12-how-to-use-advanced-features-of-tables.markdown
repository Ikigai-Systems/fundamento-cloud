Let's focus on the other possibilities of using out tables.

In order to better understand what you can transform a table for, you need to know that table consists of:

*   Rows - compose the horizontal elements of a table. Rows generally represent “things” (people names, tasks, inventory items, places to visit, etc.)

<!---->

*   Colums - thatare a table’s vertical elements, allowing you to store more information in your rows. Columns are generally “attributes” of those things (their gender, age, address, etc.).

In Fundamento, you can change column types to add various attributes to your row data. You can also reorder columns according to your preference. change column types, AI generator, buttons, sliders, people, etc. (more about it above in the Section: Property Type)

What is very important - each Column has immutable ID unique within the Table. It can be used as a reference in formulas you create.

*   Table ID - Every Table has immutable ID unique within Space. This ID can be used in formulas you create. You can review the ID value (or copy it to clipboard) in the top-right corner of the Table:

![image.png](attachment:395)

As you click on the Table ID, it will be copied to the clipboard. You can now paste it in any place just as refference without Table copy itself.
